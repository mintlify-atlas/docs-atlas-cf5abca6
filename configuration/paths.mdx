---
title: "Paths"
description: "Where Luanti stores its executables, shared data, and user files on Windows, Linux, and macOS."
---

Luanti organizes files into three logical locations:

| Location | Purpose |
|---|---|
| `bin` | Compiled executables |
| `share` | Distributed, read-only engine data (textures, sounds, built-in games) |
| `user` | User-created and modifiable data — worlds, mods, settings, screenshots |

## Platform paths

<Tabs>
  <Tab title="Windows">
    ### Installed (via installer)

    | Location | Path |
    |---|---|
    | `bin` | `C:\Program Files\Minetest\bin` (depends on install location) |
    | `share` | `C:\Program Files\Minetest` (depends on install location) |
    | `user` | `%APPDATA%\Minetest` |

    You can override the `user` path by setting the `MINETEST_USER_PATH` environment variable.

    ### Portable (.zip / run-in-place)

    | Location | Path |
    |---|---|
    | `bin` | `bin\` (relative to extracted folder) |
    | `share` | `.` (the extracted folder itself) |
    | `user` | `.` (the extracted folder itself) |

    In run-in-place mode, everything lives together in the same directory — useful for carrying Luanti on a USB drive or keeping multiple installs isolated.
  </Tab>
  <Tab title="Linux">
    ### Installed (system package)

    | Location | Path |
    |---|---|
    | `bin` | `/usr/bin` |
    | `share` | `/usr/share/minetest` |
    | `user` | `~/.minetest` |

    You can override the `user` path by setting the `MINETEST_USER_PATH` environment variable:

    ```bash
    export MINETEST_USER_PATH=/home/alice/luanti-data
    luanti
    ```
  </Tab>
  <Tab title="macOS">
    ### Application bundle

    | Location | Path |
    |---|---|
    | `bin` | `Contents/MacOS` (inside the `.app` bundle) |
    | `share` | `Contents/Resources` (inside the `.app` bundle) |
    | `user` | `Contents/User` or `~/Library/Application Support/minetest` |

    You can override the `user` path by setting the `MINETEST_USER_PATH` environment variable.
  </Tab>
</Tabs>

## Important subdirectories

All of the following paths are relative to the `user` location for your platform.

| Path | Contents |
|---|---|
| `user/worlds/` | Saved worlds — each world is a separate subdirectory |
| `user/minetest.conf` | Main configuration file |
| `user/mods/` | Globally installed mods |
| `user/games/` | Installed games |
| `user/textures/` | Texture packs |
| `user/screenshots/` | Screenshots saved with F12 (default; overridable via `screenshot_path`) |
| `user/debug.txt` | Debug log file |

## Configuration file lookup

Luanti searches for `minetest.conf` in the following order:

1. The path passed via `--config <path>` on the command line.
2. `user/minetest.conf` (the standard location).
3. For run-in-place builds: `<exe>/../minetest.conf` and `<exe>/../../minetest.conf`.

The file is created automatically the first time you close Luanti if it does not already exist.

<Note>
  The `user` directory is sometimes referred to as the "user data directory" or "user path" in older documentation and forum posts. On all platforms it can be overridden with the `MINETEST_USER_PATH` environment variable.
</Note>
