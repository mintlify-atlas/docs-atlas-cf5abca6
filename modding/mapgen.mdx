Luanti generates the world procedurally using built-in map generators (mapgens). Games and mods can configure generation by registering biomes, ores, and decorations, and by hooking into the generation pipeline with the VoxelManip API.

## Built-in mapgen algorithms

<CardGroup cols={2}>
  <Card title="v7" icon="mountain">
    The default mapgen. Produces realistic continents, oceans, mountains, rivers, and a full biome system. Recommended for most games.
  </Card>
  <Card title="v6" icon="hill">
    The classic Minetest mapgen. Simpler, with baked-in biomes (desert, tundra, etc.). Does not support the biome API.
  </Card>
  <Card title="v5" icon="wave-square">
    Older noise-based terrain without biome support.
  </Card>
  <Card title="valleys" icon="water">
    Produces lush valleys with rivers and hills.
  </Card>
  <Card title="carpathian" icon="mountain-sun">
    Dramatic alpine terrain with ridges and high peaks.
  </Card>
  <Card title="fractal" icon="snowflake">
    Generates 3D fractals (Mandelbrot, Julia sets).
  </Card>
  <Card title="flat" icon="minus">
    A completely flat world at y=0. Useful for creative games.
  </Card>
  <Card title="singlenode" icon="cube">
    An empty world filled with a single node type (default: air). Used for games that manage world generation entirely in Lua.
  </Card>
</CardGroup>

You can restrict which mapgens your game supports in `game.conf`:

```ini
# Only allow these mapgens
allowed_mapgens = v7,flat

# Or explicitly disallow some
disallowed_mapgens = v5,v6,fractal

# Set the default selection
default_mapgen = v7
```

## Mapgen aliases

Games must register aliases to tell the mapgens which nodes to use for terrain. These aliases are set with `core.register_alias()`.

### Essential aliases for all non-V6 mapgens

```lua
core.register_alias("mapgen_stone",              "default:stone")
core.register_alias("mapgen_water_source",       "default:water_source")
core.register_alias("mapgen_river_water_source", "default:river_water_source")
-- river_water_source needs a short liquid_range and liquid_renewable=false
-- to avoid flooding sloped river channels
```

### Optional aliases

```lua
core.register_alias("mapgen_lava_source", "default:lava_source")
-- Fallback cave liquid if biomes don't define node_cave_liquid

core.register_alias("mapgen_cobble", "default:cobble")
-- Fallback dungeon material if biomes don't define node_dungeon
```

### V6-specific aliases (required for Mapgen V6)

```lua
core.register_alias("mapgen_stone",           "default:stone")
core.register_alias("mapgen_water_source",    "default:water_source")
core.register_alias("mapgen_lava_source",     "default:lava_source")
core.register_alias("mapgen_dirt",            "default:dirt")
core.register_alias("mapgen_dirt_with_grass", "default:dirt_with_grass")
core.register_alias("mapgen_sand",            "default:sand")
core.register_alias("mapgen_cobble",          "default:cobble")
core.register_alias("mapgen_tree",            "default:tree")
core.register_alias("mapgen_leaves",          "default:leaves")
core.register_alias("mapgen_apple",           "default:apple")
-- Optional V6 aliases
core.register_alias("mapgen_gravel",          "default:gravel")
core.register_alias("mapgen_pine_tree",       "default:pine_tree")
core.register_alias("mapgen_pine_needles",    "default:pine_needles")
```

### Singlenode fill

```lua
-- Fill the world with stone instead of air
core.register_alias("mapgen_singlenode", "default:stone")
```

## Registering biomes

Biomes define the surface composition, water/cave node types, and placement on the heat/humidity Voronoi diagram. Register biomes with `core.register_biome()`.

<Note>
Biomes are only supported by mapgens v5, v7, valleys, carpathian, fractal, and flat. Mapgen v6 has baked-in biomes and ignores registered biomes.
</Note>

```lua
core.register_biome({
    name = "tundra",

    -- Surface layers
    node_top    = "default:dirt_with_snow",
    depth_top   = 1,                         -- 1 node thick
    node_filler = "default:permafrost",
    depth_filler = 3,                        -- 3 nodes deep
    node_stone  = "default:bluestone",       -- replaces stone in this biome

    -- Water/ice layers
    node_water_top   = "default:ice",
    depth_water_top  = 10,
    node_water       = "",                   -- uses mapgen_water_source as default
    node_river_water = "default:ice",
    node_riverbed    = "default:gravel",
    depth_riverbed   = 2,
    node_dust        = "default:snow",       -- dusted on top after generation

    -- Cave liquids
    node_cave_liquid = {"default:water_source", "default:lava_source"},

    -- Dungeon materials
    node_dungeon       = "default:cobble",
    node_dungeon_alt   = "default:mossycobble",
    node_dungeon_stair = "stairs:stair_cobble",

    -- Placement: Voronoi diagram with heat and humidity axes
    -- Heat and humidity average 50 and mostly range 0–100
    heat_point     = 5,
    humidity_point = 20,

    -- Vertical extent
    y_min = -31000,
    y_max = 31000,

    -- Blend with biome above over this many nodes
    vertical_blend = 8,
})
```

### Biome heat and humidity

Biomes are distributed on a 2D Voronoi diagram with heat (X axis) and humidity (Y axis). Each biome claims the area of the diagram closest to its `heat_point` and `humidity_point`. During generation, each map column samples the heat and humidity noise maps to determine which biome it falls in.

```lua
-- Example biome palette covering a range of climates
core.register_biome({name="hot_desert",  heat_point=95, humidity_point=5})
core.register_biome({name="savanna",     heat_point=80, humidity_point=40})
core.register_biome({name="rainforest",  heat_point=80, humidity_point=90})
core.register_biome({name="temperate",   heat_point=50, humidity_point=50})
core.register_biome({name="tundra",      heat_point=10, humidity_point=30})
core.register_biome({name="ice_plains",  heat_point=5,  humidity_point=60})
```

## Registering ores

Ores are node types scattered through stone using noise-based algorithms.

```lua
core.register_ore({
    ore_type = "scatter",      -- "scatter", "sheet", "puff", "blob", "vein", "stratum"
    ore      = "mymod:iron_ore",
    wherein  = "default:stone", -- node to replace (can be a list)

    -- Cluster density: average one cluster per 8x8x8 area
    clust_scarcity = 8 * 8 * 8,
    clust_num_ores = 5,         -- nodes per cluster
    clust_size     = 3,         -- bounding box size

    y_min = -31000,
    y_max = 64,                 -- only below y=64

    -- Optional: restrict to specific biomes
    biomes = {"temperate", "tundra"},
})

-- Sheet ore: horizontal veins of varying thickness
core.register_ore({
    ore_type          = "sheet",
    ore               = "default:sandstone",
    wherein           = "default:sand",
    column_height_min = 2,
    column_height_max = 8,
    column_midpoint_factor = 0.5,
    y_min = -15, y_max = 0,
    noise_params = {
        offset = 0, scale = 1,
        spread = {x=100, y=100, z=100},
        seed = 23, octaves = 3, persistence = 0.7,
    },
})

-- Stratum ore: simple horizontal layer
core.register_ore({
    ore_type         = "stratum",
    ore              = "mymod:basalt",
    wherein          = "default:stone",
    y_min = -500, y_max = -400,
    stratum_thickness = 20,
})
```

## Registering decorations

Decorations place nodes or schematics on top of the generated terrain.

<Tabs>
  <Tab title="Simple decoration">
    Places a single node (or random choice from a list) on a surface.

    ```lua
    core.register_decoration({
        deco_type = "simple",
        place_on  = "default:dirt_with_grass",

        -- Coverage: fill_ratio = decorations per surface node
        fill_ratio = 0.05,  -- 5 per 100 surface nodes
        -- Or use noise for variable density:
        -- noise_params = { ... }

        y_min = 1,
        y_max = 31000,

        -- Only spawn next to water
        spawn_by     = "default:water_source",
        num_spawn_by = 1,

        decoration = "default:grass",  -- or a list for random selection
        height     = 1,
        height_max = 3,  -- random height 1–3

        biomes = {"temperate", "savanna"},
    })
    ```
  </Tab>
  <Tab title="Schematic decoration">
    Places a `.mts` schematic file as a decoration (e.g. trees, buildings).

    ```lua
    local modpath = core.get_modpath("mymod")

    core.register_decoration({
        deco_type = "schematic",
        place_on  = "default:dirt_with_grass",
        fill_ratio = 0.002,
        y_min = 5,
        y_max = 31000,
        biomes = {"temperate"},

        schematic = modpath .. "/schematics/oak_tree.mts",
        -- or define inline:
        -- schematic = {
        --     size = {x=5, y=8, z=5},
        --     data = { {name="air"}, {name="default:tree"}, ... },
        -- },

        flags    = "place_center_x, place_center_z",
        rotation = "random",  -- "0", "90", "180", "270", or "random"
    })
    ```
  </Tab>
  <Tab title="L-system tree">
    Procedurally generates a tree using L-system rules.

    ```lua
    core.register_decoration({
        deco_type = "lsystem",
        place_on  = "default:dirt_with_grass",
        fill_ratio = 0.001,
        y_min = 1, y_max = 31000,
        biomes = {"temperate"},

        treedef = {
            axiom   = "TTFF A",
            rules_a = "[&FFBFA]////[&FFBFA]////[&FFBFA]",
            rules_b = "[&FFFA]////[&FFFA]",
            trunk   = "default:tree",
            leaves  = "default:leaves",
            angle   = 30,
            iterations    = 2,
            random_level  = 0,
            trunk_type    = "single",
            thin_branches = true,
        },
    })
    ```
  </Tab>
</Tabs>

## core.register_on_generated()

The `on_generated` callback fires after each mapchunk (80x80x80 nodes) is generated. Use it to post-process generated terrain.

```lua
core.register_on_generated(function(minp, maxp, blockseed)
    -- minp, maxp: corners of the generated area
    -- blockseed: deterministic seed for this chunk

    -- Spawn structures at specific conditions
    if math.random(1, 100) > 95 then
        local center = {
            x = math.floor((minp.x + maxp.x) / 2),
            y = math.floor((minp.y + maxp.y) / 2),
            z = math.floor((minp.z + maxp.z) / 2),
        }
        -- Find a suitable surface position and place something
    end
end)
```

<Warning>
Avoid using `core.register_on_generated` when possible — it blocks the main thread and causes lag. For bulk node manipulation, use the VoxelManip API (see below) or the Mapgen environment instead.
</Warning>

## The VoxelManip API

VoxelManip provides fast, bulk read/write access to map data. It is the right tool for custom mapgen operations that touch many nodes at once.

### Creating a VoxelManip

```lua
-- Create and load a region of the map
local vm = VoxelManip()
local emin, emax = vm:read_from_map(minp, maxp)
-- emin/emax are the actual loaded region (may be larger than requested)

-- Or create with pre-loaded region
local vm = VoxelManip(minp, maxp)
local emin, emax = vm:get_emerged_area()

-- Inside on_generated, reuse the engine's VoxelManip
local vm = core.get_mapgen_object("voxelmanip")
local emin, emax = vm:get_emerged_area()
```

### Reading and writing node data

```lua
-- Get bulk node content IDs
local data = vm:get_data()       -- flat array of content IDs
local p2data = vm:get_param2_data()  -- flat array of param2 values

-- Use VoxelArea to calculate flat array indices
local va = VoxelArea:new({MinEdge=emin, MaxEdge=emax})

-- Read a single node
local idx = va:index(x, y, z)
local content_id = data[idx]
local name = core.get_name_from_content_id(content_id)

-- Write a single node
local stone_id = core.get_content_id("default:stone")
local c_air    = core.CONTENT_AIR
data[va:index(x, y, z)] = stone_id

-- Built-in content ID constants
-- core.CONTENT_AIR     -- ID for air
-- core.CONTENT_IGNORE  -- ID for ignore (unloaded)
-- core.CONTENT_UNKNOWN -- ID for unknown nodes

-- Flush modified data back to the VoxelManip's internal buffer
vm:set_data(data)
vm:set_param2_data(p2data)

-- Write the VoxelManip back to the map
vm:write_to_map()
-- Pass false to skip lighting update (faster, but needs manual calc_lighting)
vm:write_to_map(false)
vm:calc_lighting()   -- recalculate light after write
vm:update_liquids()  -- start liquid flow for any liquid nodes added
```

### Complete on_generated example with VoxelManip

```lua
local c_stone   = core.get_content_id("default:stone")
local c_glass   = core.get_content_id("default:glass")
local c_air     = core.CONTENT_AIR

core.register_on_generated(function(minp, maxp, blockseed)
    -- Only process chunks that include y=0 and are near origin
    if minp.y > 0 or minp.x > 100 or minp.z > 100 then
        return
    end

    local vm   = core.get_mapgen_object("voxelmanip")
    local emin, emax = vm:get_emerged_area()
    local data = vm:get_data()
    local va   = VoxelArea:new({MinEdge=emin, MaxEdge=emax})

    -- Replace all stone in this chunk with glass
    for z = minp.z, maxp.z do
        for y = minp.y, maxp.y do
            for x = minp.x, maxp.x do
                local idx = va:index(x, y, z)
                if data[idx] == c_stone then
                    data[idx] = c_glass
                end
            end
        end
    end

    vm:set_data(data)
    vm:write_to_map()
end)
```

<Note>
For a 3x3x3 node region or smaller, `core.set_node()` in a loop is actually faster than VoxelManip. VoxelManip becomes advantageous for larger areas.
</Note>

### Other VoxelManip operations

```lua
-- After writing liquid nodes, trigger liquid flow
vm:update_liquids()

-- Generate registered ores and decorations inside the VoxelManip region
core.generate_ores(vm)
core.generate_decorations(vm)

-- Place a schematic into the VoxelManip (does not write to map)
core.place_schematic_on_vmanip(vm, pos, schematic, rotation, replacements, force_placement)

-- Lighting
vm:calc_lighting()         -- full lighting recalculation
vm:set_lighting({day=15, night=0}, minp, maxp)  -- set uniform lighting
```

## Mapgen environment

For performance-critical custom generation, Luanti supports a dedicated mapgen environment. Callbacks registered with `core.register_mapgen_callback()` run in a separate thread with access to a restricted subset of the API, avoiding the main thread overhead of `on_generated`.

The mapgen environment can access:

```lua
-- Available in mapgen environment (subset of full API)
core.get_node, core.set_node, core.find_node_near,
core.find_nodes_in_area, core.get_mapgen_object,
core.generate_ores, core.generate_decorations,
-- (among others; see the API reference for the full list)
```

Full player and world interaction APIs (`core.get_player_by_name`, etc.) are not available in the mapgen environment.

## Clearing and re-registering

If your game needs to reset its mapgen content (e.g. to hot-reload), use the clear functions:

```lua
core.clear_registered_biomes()
core.clear_registered_decorations()
core.clear_registered_ores()
core.clear_registered_schematics()

-- Warning: clearing biomes invalidates biome IDs used in ore and decoration
-- 'biomes' fields, so you must re-register those too.
```
