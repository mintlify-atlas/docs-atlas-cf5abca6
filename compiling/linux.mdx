## Dependencies

The table below lists all required and optional build dependencies.

| Dependency | Version | Notes |
|---|---|---|
| GCC | 7.5+ | Clang 7.0.1+ also works |
| CMake | 3.5+ | |
| libjpeg | — | |
| libpng | — | |
| SDL | 2.x | |
| Freetype | 2.0+ | |
| SQLite3 | 3+ | |
| Zlib | — | |
| Zstd | 1.0+ | |
| LuaJIT | 2.0+ | Bundled Lua 5.1 is used if not present |
| GMP | 5.0.0+ | Bundled mini-GMP is used if not present |
| JsonCPP | 1.0.0+ | Bundled JsonCPP is used if not present |
| Curl | 7.56.0+ | Optional |
| gettext | — | Optional |
| OpenSSL | 3.0+ | Optional — only `libcrypto` is used |

## Build

Build a version that runs directly from the source directory:

```bash
cmake . -DRUN_IN_PLACE=TRUE
make -j$(nproc)
```

Run Luanti:

```bash
./bin/luanti
```

Run unit tests to verify your build:

```bash
./bin/luanti --run-unittests
```

## Common build variations

| Goal | CMake flag |
|---|---|
| System-wide install (e.g. for packaging) | `-DRUN_IN_PLACE=FALSE` |
| Build a dedicated server only | `-DBUILD_SERVER=TRUE -DBUILD_CLIENT=FALSE` |
| Debug build (slower but better for debuggers) | `-DCMAKE_BUILD_TYPE=Debug` |
| See all available options | `cmake . -LH` |

<Tip>
  Use `-DCMAKE_BUILD_TYPE=Debug` when you want to attach a debugger or investigate crashes. Debug builds produce much more useful output, but run noticeably slower than release builds.
</Tip>

For a full list of CMake options, see the [compiling overview](/compiling/overview).
