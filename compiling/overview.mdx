## CMake options

Pass options to CMake using the `-D<OPTION>=<VALUE>` syntax:

```bash
cmake . -DBUILD_SERVER=TRUE -DCMAKE_BUILD_TYPE=Release
```

To list all available options and their current values:

```bash
cmake . -LH
```

### General build options

| Option | Default | Description |
|---|---|---|
| `BUILD_CLIENT` | `TRUE` | Build the Luanti client |
| `BUILD_SERVER` | `FALSE` | Build the Luanti server |
| `BUILD_UNITTESTS` | `TRUE` | Build unit test sources |
| `BUILD_BENCHMARKS` | `FALSE` | Build benchmark sources |
| `BUILD_DOCUMENTATION` | `TRUE` | Build Doxygen documentation |
| `CMAKE_BUILD_TYPE` | `Release` | Build type: `Release`, `Debug`, `SemiDebug`, `RelWithDebInfo`, or `MinSizeRel` |
| `PRECOMPILE_HEADERS` | `FALSE` | Precompile some headers (experimental; requires CMake 3.16+) |
| `PRECOMPILED_HEADERS_PATH` | _(auto)_ | Path to the file listing headers to precompile |
| `USE_SDL2_STATIC` | `TRUE` | Link with `SDL2::SDL2-static` instead of `SDL2::SDL2` |
| `USE_SDL3` | `FALSE` | Build with SDL3 (experimental) |
| `USE_SDL3_STATIC` | `TRUE` | Link with `SDL3::SDL3-static` instead of `SDL3::SDL3-shared` |
| `RUN_IN_PLACE` | `FALSE` | Create a portable install — worlds and settings live in the current directory |
| `ENABLE_UPDATE_CHECKER` | `TRUE` | Enable update checks by default |
| `INSTALL_DEVTEST` | `FALSE` | Install the Development Test game alongside Luanti |
| `USE_GPROF` | `FALSE` | Enable profiling with GProf |
| `BUILD_WITH_TRACY` | `FALSE` | Fetch and build with the Tracy profiler client |
| `FETCH_TRACY_GIT_TAG` | `master` | Git tag for fetching the Tracy client |
| `VERSION_EXTRA` | _(empty)_ | Text appended to the version string (e.g. `foobar` → `Luanti 5.10.0-foobar`) |

### Feature flags

| Option | Default | Description |
|---|---|---|
| `ENABLE_CURL` | `ON` | Build with cURL — enables server list, content browser, and more |
| `ENABLE_CURSES` | `ON` | Build with (n)curses — enables server-side terminal (`--terminal`) |
| `ENABLE_GETTEXT` | `ON` | Build with Gettext — enables translations |
| `ENABLE_LEVELDB` | `ON` | Build with LevelDB — enables the LevelDB map backend |
| `ENABLE_POSTGRESQL` | `ON` | Build with libpq — enables the PostgreSQL map backend (9.5+ recommended) |
| `ENABLE_REDIS` | `ON` | Build with libhiredis — enables the Redis map backend |
| `ENABLE_SPATIAL` | `ON` | Build with LibSpatial — speeds up AreaStores |
| `ENABLE_OPENSSL` | `ON` | Build with OpenSSL — speeds up SHA1 and SHA2 hashing |
| `ENABLE_SOUND` | `ON` | Build with OpenAL, libogg, and libvorbis — enables in-game sounds |
| `ENABLE_LTO` | _(varies)_ | Build with IPO/LTO optimizations — smaller and faster than a regular build |
| `ENABLE_LUAJIT` | `ON` | Build with LuaJIT (much faster than the non-JIT Lua interpreter) |
| `ENABLE_PROMETHEUS` | `OFF` | Build with Prometheus metrics exporter (listens on TCP 30000 by default) |
| `ENABLE_SYSTEM_GMP` | `ON` | Use GMP from the system (much faster than the bundled mini-GMP) |
| `ENABLE_SYSTEM_JSONCPP` | `ON` | Use JsonCPP from the system |

## Build types

The `CMAKE_BUILD_TYPE` option controls optimization level and debug information:

| Value | Description |
|---|---|
| `Release` | Fully optimized production build (default) |
| `Debug` | No optimization; maximum debug info — use this when debugging with a debugger |
| `SemiDebug` | Partially optimized debug build |
| `RelWithDebInfo` | Release optimization with debug symbols included |
| `MinSizeRel` | Release build with `-Os` to minimize executable size |

<Tip>
  Use `Debug` builds when reporting issues or profiling with a debugger. Debug builds produce much more useful output but run noticeably slower.
</Tip>
